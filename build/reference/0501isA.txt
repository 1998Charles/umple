isA clause
Classes and Interfaces

@@description
<p>Used to denote an inheritance relationship (generalization) between two classes</p>

<p>This corresponds to keywords such as 'extends', 'subclass', etc. in other languages. The isA keyword was chosen so as to be independent of other languages.</p>

@@syntax
isA [extendsName] ;

@@example

class Person
{
  name;
}

class Student
{
  isA Person;
  Integer number;
}//$?[End_of_model]$?

class Person
{
  position 45 21 113 63;
}

class Student
{
  position 50 141 135 63;
}
@@endexample
